This is one of my passion projects which is now being used by Kohl's developers

This project was created during my internship with Kohl's. The idea for the project came when my mentor was explaining an issue that the team was having when using Splunk. This application opens multiple Splunk URLs and runs the desired searches based on which button is pushed. A Store, Register, and Transaction number can be used to allow for a more specific search. URLs can be added, removed, adjusted, and viewed. Buttons can be renamed. The application will save all changes to a text file. When the application is reopened, all of the previous modifications will still be present.

Specifics about the project

Below are three examples where I faced a problem and created a solution

Saving changes

Changes such as adding, altering, and removing URLS from buttons and renaming buttons needed to be saved. With a limited amount of time I decided to use a text file.

Using variables with a URL

Since some URLs utilized one or more variables, it was important to keep that in mind when saving to a text file and running a search.

variable formatting

The Store, Register, and Transaction numbers all had different formatting requirements when running a splunk search.
